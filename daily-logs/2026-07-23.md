# 세션 2 — 관리자 로그인 + 소스 보호(난독화)

## 한 일

### 1. 관리자 페이지 로그인 (`/admin/`)
- 비밀번호 로그인 화면 추가 — 입장 전 로그인 필수.
- 비밀번호는 코드에 저장되지 않고 **Apps Script의 `ADMIN_KEY`와 서버에서 대조** (auth 엔드포인트 신설).
- 로그인 성공 시 sessionStorage에 키 보관(탭 닫으면 소멸) + 자동 재로그인, 로그아웃 버튼.
- 서버 보호 강화(fail-closed): `ADMIN_KEY` 미설정 시 로그인·등록자 조회·코스 저장 전부 거부.
  `?all=1`(숨김 코스 포함 조회)도 키 필수로 변경. 공개 코스 목록(등록 폼용)은 그대로 공개.

### 2. 소스 분석 억제 (경쟁사 대응)
- **프로덕션 번들 난독화** (vite.config.ts + javascript-obfuscator):
  문자열 전부 base64 배열로 은닉(EN/KR 카피 포함), 식별자 16진수화, split-strings, 콘솔 출력 제거, 소스맵 없음.
  번들 781KB(gzip 238KB)로 증가 — 성능 큰 영향 없는 수준. controlFlowFlattening은 성능 저하로 미적용.
- **개발자도구 억제** (src/protect.ts, 프로덕션+실도메인에서만 작동):
  우클릭 차단, 이미지 드래그 저장 차단, F12·Ctrl/Cmd+Shift+I/J/C·Ctrl/Cmd+U·Ctrl/Cmd+S 차단.
- Playwright(크로미움)로 검증: 렌더링 정상(8개 섹션·#register 딥링크 OK), 실도메인 조건에서 차단 작동 확인.

## ⚠️ 한계 & 필수 후속 조치 (사용자 작업)
1. **Apps Script에 `ADMIN_KEY` 설정 + 새 버전 재배포 필수** — 안 하면 이 변경 배포 후 /admin/ 로그인이 거부됨(잠김).
   Apps Script 편집기에서 `var ADMIN_KEY = '원하는비밀번호';` 설정 → 저장 → 배포 관리 → 새 버전.
2. 브라우저는 코드를 항상 받아야 하므로 **100% 은닉은 기술적으로 불가능** — 난독화+차단은 억제 수단.

# 세션 2b — ⚠️ 저장소 private 전환 사고 & 복구 (중요 교훈)

## 무슨 일이 있었나
- 소스 은닉을 위해 사용자가 **저장소를 Private으로 전환** → **사이트 전체 다운(403)**.
  원인: 이 저장소는 **무료 플랜**이라 private로 바꾸면 GitHub Pages 기능이 즉시 꺼진다.
  (private 저장소에서 Pages 서빙하려면 GitHub Pro 등 유료 플랜 필요.)
- 다시 **public으로 되돌려도 자동 복구 안 됨** — Pages가 "disabled" 상태로 남는다.

## 복구 절차 (이 순서대로 하면 됨 — 다음에 또 막히면 참고)
1. 저장소 Settings → Danger Zone → **Change visibility → Public**.
2. Settings → **Pages → Build and deployment → Source**(위쪽 드롭다운)를 **"GitHub Actions"** 로 재선택.
   (아래 "None" 드롭다운은 브랜치 선택용이라 헷갈리지 말 것. Custom domain `letspilatesla.com` 유지 확인.)
3. 재배포 트리거: Staging에 빈 커밋 push (커밋 aa9ca8c) → 워크플로우가 root+staging 전체 재배포.
4. 몇 분 뒤 https://letspilatesla.com/ 정상 확인 (인증서 재발급에 수 분 걸릴 수 있음). → **복구 완료 확인됨.**

## 결론 / 방침 변경
- **무료 플랜에서는 "저장소 private + GitHub Pages"가 불가능하다.** 이전 로그의 "private 전환 권장"은 철회.
- 소스 비공개 + 호스팅 흔적 은닉을 둘 다 원하면 → **Cloudflare Pages 이전**이 답 (무료, private 저장소 지원, github.io 주소도 사라짐). 이번엔 사용자가 "지금은 여기까지" 선택 → 보류.
- 참고: Claude 실행 환경의 프록시가 letspilatesla.com/letspilates.github.io 접속을 정책 차단 →
  Claude는 라이브 사이트를 직접 curl로 못 연다. 배포 검증은 GitHub Actions 실행 결과 + 사용자 브라우저 확인으로 한다.
