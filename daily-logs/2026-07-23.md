# 세션 2 — 관리자 로그인 + 소스 보호(난독화)

## 한 일

### 1. 관리자 페이지 로그인 (`/admin/`)
- 비밀번호 로그인 화면 추가 — 입장 전 로그인 필수.
- 비밀번호는 코드에 저장되지 않고 **Apps Script의 `ADMIN_KEY`와 서버에서 대조** (auth 엔드포인트 신설).
- 로그인 성공 시 sessionStorage에 키 보관(탭 닫으면 소멸) + 자동 재로그인, 로그아웃 버튼.
- 서버 보호 강화(fail-closed): `ADMIN_KEY` 미설정 시 로그인·등록자 조회·코스 저장 전부 거부.
  `?all=1`(숨김 코스 포함 조회)도 키 필수로 변경. 공개 코스 목록(등록 폼용)은 그대로 공개.

### 2. 소스 분석 억제 (경쟁사 대응)
- **프로덕션 번들 난독화** (vite.config.ts + javascript-obfuscator):
  문자열 전부 base64 배열로 은닉(EN/KR 카피 포함), 식별자 16진수화, split-strings, 콘솔 출력 제거, 소스맵 없음.
  번들 781KB(gzip 238KB)로 증가 — 성능 큰 영향 없는 수준. controlFlowFlattening은 성능 저하로 미적용.
- **개발자도구 억제** (src/protect.ts, 프로덕션+실도메인에서만 작동):
  우클릭 차단, 이미지 드래그 저장 차단, F12·Ctrl/Cmd+Shift+I/J/C·Ctrl/Cmd+U·Ctrl/Cmd+S 차단.
- Playwright(크로미움)로 검증: 렌더링 정상(8개 섹션·#register 딥링크 OK), 실도메인 조건에서 차단 작동 확인.
