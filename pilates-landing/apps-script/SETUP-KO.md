# 코스 목록 관리 설정 가이드 (한 번만 하면 됩니다)

이 설정을 마치면 **구글 시트에서 코스를 수정/추가/삭제하는 것만으로 사이트 등록 폼에 바로 반영**됩니다.
코드나 배포를 다시 만질 일은 없습니다.

소요 시간: 약 5분

---

## 1단계 — Apps Script 코드 교체

등록 신청이 쌓이는 **기존 "Registration" 스프레드시트**를 엽니다.

1. 메뉴 **확장 프로그램 → Apps Script** 를 엽니다.
2. 편집기에 있는 기존 코드를 **전부 지우고**, 이 폴더의 `google-apps-script.js` 파일 내용을 **전체 복사해서 붙여넣습니다.**
3. 💾 저장 (Ctrl+S / Cmd+S)

## 2단계 — Courses 탭 자동 생성 (클릭 한 번)

수동으로 표를 붙여넣을 필요 없습니다. 스크립트가 대신 만들어 줍니다:

1. Apps Script 편집기 **상단 툴바의 함수 선택 드롭다운** (▶ 실행 버튼 옆)에서 **`setupCoursesTab`** 을 선택합니다.
2. **▶ 실행** 버튼을 클릭합니다.
3. 처음이면 권한 요청 창이 뜹니다 → **권한 검토 → 계정 선택 → 고급 → (안전하지 않음) 이동 → 허용**
   (본인 계정의 본인 스크립트라 안전합니다. 구글이 미검증 스크립트에 항상 띄우는 경고입니다.)
4. 실행이 끝나면 스프레드시트로 돌아가 보세요 — **"Courses" 탭이 기본 코스 5개와 함께 생성**되어 있습니다.
   (dates 열 텍스트 서식, 헤더 굵게, 행 고정까지 자동으로 되어 있습니다)

### 각 열의 의미

| 열 | 의미 | 예시 |
|---|---|---|
| `id` | 코스 구분 문자 (원 안에 표시됨) | A, B, C... |
| `name_en` | 영문 코스명 | GYROTONIC® Level 1 Foundation Course |
| `name_kr` | 한글 코스명 | GYROTONIC® Level 1 기초 과정 |
| `dates` | 일정 (비워두면 "일정 추후 공지"로 표시) | 4/10–12, 4/17–19 |
| `tag_en` | 영문 기간 뱃지 | 12 days |
| `tag_kr` | 한글 기간 뱃지 | 12일 |
| `active` | TRUE = 사이트에 표시, FALSE = 숨김 | TRUE |
| `capacity` | 정원 (숫자). 입력하면 사이트에 "N자리 남음"이 표시되고, 신청이 정원에 차면 자동으로 "마감" 처리됨. **비워두면 자리 표시 없이 항상 신청 가능** | 8 |

> **남은 자리 계산 방식**: 접수 시트(첫 탭)에 쌓인 신청 수를 코스별로 세서 `정원 − 신청 수`로 자동 계산합니다.
> 새 기수를 시작할 때는 이전 신청 행들을 다른 탭(예: "2026 접수분")으로 옮겨두면 카운트가 0부터 다시 시작됩니다.

---

## 3단계 — 재배포 (⚠️ 가장 중요 — "새 배포" 아님!)

주소가 바뀌면 사이트와 연결이 끊어지므로, 반드시 **기존 배포를 업데이트**해야 합니다:

1. 우측 상단 **배포 → 배포 관리** 클릭
2. 기존 배포(웹 앱) 옆의 **✏️ 연필(수정) 아이콘** 클릭
3. **버전** 드롭다운에서 **"새 버전"** 선택
4. **배포** 클릭

> ❌ "새 배포" 버튼을 누르면 새 주소가 생겨서 사이트와 연결이 끊깁니다.
> 반드시 "배포 관리 → 연필 → 새 버전" 순서로 해주세요.

## 4단계 — 확인

브라우저 새 탭에서 기존 웹 앱 주소(`https://script.google.com/macros/s/.../exec`)를 열었을 때
아래처럼 **코스 목록 JSON**이 보이면 성공입니다:

```json
{"result":"success","courses":[{"id":"A","name_en":"GYROTONIC® Level 1 Foundation Course",...}]}
```

이후 사이트(스테이징 또는 라이브)에서 폼을 열면 시트의 목록이 그대로 표시됩니다.

## 이메일 알림 설정 (한 번만)

새 코드에는 **신청이 들어올 때마다 이메일 알림**이 포함되어 있습니다.

1. 함수 드롭다운에서 **`sendTestEmail`** 선택 → **▶ 실행**
2. 처음 실행하면 **이메일 발송 권한 승인 창**이 뜹니다 → 허용
   (⚠️ 이 승인을 안 하면 실제 신청 때 알림이 안 갑니다 — 꼭 한 번 실행하세요)
3. 받은편지함에 테스트 메일이 오면 성공. 이후 모든 신청이 이메일로 옵니다.

- 기본 수신자: 시트 소유자 계정 (calvin3919@gmail.com)
- 다른 주소로 받으려면: 코드 맨 위 `var NOTIFY_EMAIL = '';` 의 따옴표 안에 이메일 입력 후 저장 → 재배포(배포 관리 → 연필 → 새 버전)

## 접수 시트 헤더 만들기 (헤더가 없거나 지워진 경우)

함수 드롭다운에서 **`setupSubmissionHeaders`** 실행 → 접수 시트(첫 탭) 1행에
한글 컬럼 헤더(제출시각 | 신청 과정 | 이름 | ...)가 자동 생성됩니다.
기존에 접수된 신청 행이 있으면 그 위에 헤더 행을 끼워 넣으므로 데이터는 보존됩니다.

## 기존 설치에서 업그레이드하는 경우

이미 이전 버전으로 Courses 탭을 만들었다면:
1. Apps Script 코드를 새 버전으로 전체 교체 → 저장
2. 함수 드롭다운에서 **`upgradeCoursesTab`** 실행 → capacity 열이 자동 추가됨
3. **`setupSubmissionHeaders`** 실행 → 접수 시트 헤더 생성
4. **`sendTestEmail`** 실행 → 이메일 권한 승인
5. **배포 관리 → 연필 → 새 버전** 재배포

## 신청자 웰컴 이메일 (자동 발송 · 영어)

신청이 접수되면 **신청자에게 사이트 테마의 HTML 웰컴 이메일이 영어로** 자동 발송됩니다.
내용: 신청 프로그램(설명·일정·시간·기간·가격) + 결제 안내 + 위치/주차 + 취소 정책.

**✏️ 이메일 내용은 두 곳에서 관리합니다:**

| 내용 | 수정하는 곳 |
|---|---|
| 결제 안내 · 주차 안내 · 취소 정책 · 주소/연락처 | **Apps Script 코드 상단 `var STUDIO = { ... }` 블록** — 수정 후 저장 + "배포 관리 → 연필 → 새 버전" 재배포 필요 |
| 코스별 시간 · 가격 · 설명 | **관리자 페이지(letspilatesla.com/admin/)** — 저장하면 즉시 적용, 재배포 불필요 |

**발신 주소 변경 (선택):**
- 기본 발신: 시트 소유 계정(calvin3919@gmail.com).
- 다른 주소로 발신하려면: 그 주소를 Gmail **설정 → 계정 → "다른 주소에서 메일 보내기"**에
  별칭으로 등록·인증한 뒤, 코드 상단 `SENDER_ALIAS`에 입력. (별칭 발신은 처음 실행 시 Gmail 권한 승인 필요)
- 답장만 다른 주소로 받으려면: `REPLY_TO`에 입력 (별칭 등록 불필요, 바로 적용).

**⚠️ 설정 시 반드시 할 일:**
1. `STUDIO` 블록의 **payment / parking / cancel** 문구를 실제 정책으로 수정 (기본값은 임시 문구!).
2. 관리자 페이지에서 코스별 **시간·가격·설명(영어)** 입력. 비워두면 이메일에서 해당 줄이 생략됩니다.
3. 함수 드롭다운에서 **`upgradeCoursesTab`** 실행 → 새 열(time/price/desc) 헤더 자동 추가.
4. **`sendTestWelcomeEmail`** 실행 → 관리자 메일함으로 샘플 웰컴 이메일이 오는지 확인.
5. **배포 관리 → ✏️ → 새 버전** 재배포.

## 관리자 페이지 설정 (letspilatesla.com/admin/)

구글 시트에 직접 들어가지 않고 **웹 관리자 페이지에서 코스를 편집**할 수 있습니다.
저장하면 시트(Courses 탭)에 기록되고 사이트에 즉시 반영됩니다.

https://letspilatesla.com/admin/ 접속 → 편집 → 저장. 비밀번호 없이 바로 사용합니다.

- 관리 항목: 코스명(영/한), 일정, 기간(일수 → "N days"/"N일" 자동 표기), 정원, 시간,
  코스 가격(Course Cost), 스튜디오 Fee, 진행자(Conducted by), 설명(이메일용).
- 신청 수/남은 자리는 자동 집계라 보기만 됩니다.
- 페이지는 검색엔진에 노출되지 않게(noindex) 설정되어 있습니다.
- 원하면 코드 상단 `ADMIN_KEY`에 비밀번호를 설정해 저장을 잠글 수 있습니다 (비워두면 비밀번호 없음).

## 이후 일상 관리 (이것만 하면 됩니다)

- **날짜 변경**: Courses 탭에서 dates 칸 수정 → 저장. 끝.
- **정원 설정/변경**: capacity 칸에 숫자 입력. 차면 사이트에 자동 "마감" 표시.
- **코스 잠시 숨기기**: active를 `FALSE`로. 다시 열 땐 `TRUE`.
- **새 코스 추가**: 새 행 추가 (id는 겹치지 않게).
- 반영 시점: 저장 즉시. (방문자 브라우저에 이전 목록이 캐시되어 있으면 다음 방문 때 갱신)

## 만약 시트/스크립트에 문제가 생기면?

사이트 폼은 절대 죽지 않습니다. 시트를 읽지 못하면 코드에 내장된 기본 코스 목록을
자동으로 표시하고, 마지막으로 성공했던 목록이 방문자 브라우저에 저장되어 있으면 그걸 먼저 씁니다.
